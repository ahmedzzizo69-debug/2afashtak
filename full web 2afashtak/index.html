Database Architecture (3NF)

Normalized schema following project requirements.

Tables
Normalization
  • Users: UserID, FullName, Email, Phone...
  • Categories: CategoryID, CategoryName
  • Locations: LocationID, LocationName, Address
  • Items: ItemID, UserID, CategoryID, LocationID, Type, Status...
  • Matches: MatchID, LostItemID, FoundItemID, Date...
3NF Compliant

The database is designed in 3rd Normal Form (3NF) to ensure data integrity and eliminate redundancy:

  • 1NF: Atomic values, no repeating groups.
  • 2NF: No partial functional dependencies.
  • 3NF: No transitive dependencies.
